How it works

From first sync to a tested recovery plan in four steps.

01

Connect a source

Add a physical host, VM cluster, or cloud account through the HyperBDR console.

02

Configure protection

Set RPO targets and retention policy; replication starts automatically.

03

Test failover

Run an isolated recovery drill in the cloud without touching production.

04

Recover or migrate

Trigger one-click recovery, or promote the target as a permanent migration.
